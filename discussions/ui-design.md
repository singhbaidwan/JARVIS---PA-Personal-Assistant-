TAB 1: Dashboard
Functional Description:
This page serves as a full-screen control center summary, providing a bird's-eye view of your entire system. It combines key elements from multiple modules:

System Core: Visualizes the connection status to your local Core server via holographic server racks.

Holographic AI: Your conversational assistant avatar.

System Health Globe: A summarizing 3D globe showing CPU, RAM, and Disk metrics.

Conversational Panel: Real-time chat history with the assistant.

Learning Insights: Quick view of behavior learning (e.g., focus time, productivity score).

Workflows: Quick access to existing workflows and the workflow runtime log.

TAB 2: System Health
Functional Description:
A dedicated page focusing on exhaustive system monitoring and health diagnostics.

Holographic Globe: The detailed centerpiece (from the summary panel) displaying intricate network and processing paths, showing current load.

Expanded Real-time Graphs: Multiple detailed timeline charts for individual CPU cores, RAM load per stick, individual Disk I/O speeds, and comprehensive Network bandwidth usage.

Battery Health & History: Deep-dive analysis of battery condition, cycle count, and usage over time.

Guardian Scan History: Lists past scans with time and result.

Critical Events Log: High-priority warning list.

Control Button: "RUN FULL SYSTEM SCAN" forces a complete diagnostic check.

Maintains overall excellent health status text.

TAB 3: Knowledge
Functional Description:
This page visualizes the vast, intricate network of information and data JARVIS has learned and is connected to.

Intricate Knowledge Graph: A massive, full-screen interactive visualization (replacing the AI head) that resembles a neural network. Specific node types define "Users," "Files," "Projects," "APIs," and "Learned Behaviors," with connecting lines showing relationships.

Knowledge Sources: A detailed panel listing all connected APIs and services (e.g., Core, AI, Slack, VS Code) with data usage statistics and connection status details, featuring edit icons.

Learned Concepts & Proposals: Displays summary models of learned user behaviors and proposed rule automated workflows for approval.

Query Input: A direct input box to specifically search or query the knowledge base.

Maintains text like "Learned Systems: Active".

TAB 4: Workflows
Functional Description:
A dedicated full-page visual workflow studio workspace.

Massive Workflow Builder Canvas: A large, intricate full-screen interactive node diagram (like the example, but significantly larger and more complex) for complex workflow creation. A side palette provides numerous node types for logic, conditions, and actions.

Comprehensive Workflow Library: A detailed, list-based library on the left with icons, names, and a context menu for editing, copying, and deleting existing workflows.

Create Button: The prominent "CREATE NEW WORKFLOW" button is located here.

Workflow Execution History: A large log panel tracking the full execution history and error details for all workflows.

Maintains workflow-specific text and the example log from the dashboard summary.

TAB 5: Settings
Functional Description:
The main configuration panel for application-level control.

Structured Configuration: A clean, multi-section layout for organizing settings.

Sections:

Application Settings: Theme (Dark/Light), Launch at Startup toggle.

Account & Integrations: Connect/Disconnect linked services (Slack, VS Code, Google Calendar).

Data & Privacy: Toggles for Local Processing mode (local-first by default), Data Export, and Data Deletion.

Core & AI Connection: Expanded inputs for Core and AI URLs, adding specific port configuration and detailed connection status logs.

Advanced: LLM Provider selection (Local vs. specific Cloud options), Voice Assistant configuration, API Access Keys.

Control Buttons: "SAVE CHANGES" is prominent, next to "Restore Defaults".

*Maintains text like "Data stream active. Local processing mode." for consistency.*
